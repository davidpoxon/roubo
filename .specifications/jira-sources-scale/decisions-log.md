# Decisions Log: jira-sources-scale

- **2026-06-01** — "Assigned to me" preset scope → offered as a per-source option with two modes (in selected Jira project, or anywhere). Rationale: covers both single-project focus and cross-project workflows without a separate source type. (captured during /product-prd)
- **2026-06-01** — Board source default → active sprint only, with an option to widen to the whole board. Rationale: matches "developers work from their scrum board" and drops backlog/done noise by default. (captured during /product-prd)
- **2026-06-01** — Picker search performance target → p95 < 500ms with ~250ms debounce, all queries scoped + paginated. Rationale: type-ahead must feel instant even on the large instance. (captured during /product-prd)
- **2026-06-01** — Accessibility bar → WCAG 2.1 AA, fully keyboard-navigable, React Aria Components. Rationale: matches Roubo's existing interaction convention and the picker is a complex composite control. (captured during /product-prd)
- **2026-06-01** — Default excluded statuses → exclude Jira's "Done" status category (category-based), user-editable. Rationale: robust to custom status names across Workday's many projects, unlike a hardcoded name list. (captured during /product-prd)
- **2026-06-01** — Picker result labels → show full name + Jira project key + underlying item id (monospace). Board type and owner explicitly NOT required. Rationale: project key + id are sufficient to disambiguate and self-serve. (captured during /product-prd)
- **2026-06-01** — Security → reuse existing PAT slot + per-instance host constraint; no logging of JQL/PAT/issue contents; add JQL-injection hardening on search terms. Rationale: new search endpoints introduce user-supplied input into JQL. (captured during /product-prd)
- **2026-06-01** — Overall t-shirt size → Small (4 sprints / 8 person-weeks). Rationale: bounded to the Jira plugin + picker contract, reuses async-search + override patterns; cascade design + spike keep it above X-Small. (captured during /product-prd)
- **2026-06-01** — Deferred work: move source category→kind mapping out of the hardcoded `CATEGORY_TO_KIND` (`plugin-source-translation.ts:36`) into the plugin manifest → kept the host edit for this Small slug; tracked separately in GitHub issue #349. Rationale: doing it mid-feature would expand scope beyond Small. (captured during /product-architecture)
- **2026-06-11** — FR-026 incremental `updated >=` watermark removed from the Jira plugin. `listIssues` is now a point-in-time fetch on every call, matching the GitHub/GHE plugin contract. Rationale: there is no accumulating issue store; the route returns items straight to the client and the cut list re-fetches on every poll. The watermark could only hide issues after the first complete page-walk, collapsing a full active sprint to a single recently-touched ticket. The `state` credential slot (state-store.ts) is also removed. The PRD/architecture FR-026 language is superseded by this decision.
